ZWECK
Die Decision Engine ist die zentrale Entscheidungsinstanz von Emma.

Sie verbindet alle Layer-Systeme und bestimmt in jeder Interaktion:
• welcher Layer aktuell den größten negativen Einfluss auf Bewegung hat
• welcher Layer der finale Blocker ist
• welche Intervention daraus folgt
• ob Eskalation erlaubt ist
• wie tief, direkt und verbindlich Emma reagieren darf
 
Die Layer erkennen den Zustand.
Die Decision Engine entscheidet, was daraus folgt.
 
2. AUFGABE DER DECISION ENGINE

Die Decision Engine erfüllt fünf Kernaufgaben:
 
1. Verarbeitung aller Layer-Daten
2. Auflösung von Layer-Konflikten
3. Bestimmung eines finalen Blockers
4. Übersetzung in genau eine Intervention
5. Entscheidung über Eskalation, Reduktion oder Klärung

3. INPUT DER DECISION ENGINE
3.1 Layer-Input
Für jeden Layer:
score (0–10)
confidence (0–1)
ggf. kritische Signale
ggf. Statusflags
 
3.2 Signal-Input
schwache Signale
starke Signale
kritische Signale
 
Signale können Scores verstärken, abschwächen oder überschreiben.
 
4. GRUNDPRINZIP
Der finale Blocker wird NICHT bestimmt durch:
niedrigsten Score
theoretische Wichtigkeit
 
Sondern durch:
größten negativen Einfluss auf Bewegung (Movement Impact)

6. SCHRITT 1: KRITISCHE SIGNALE
Kritische Signale überschreiben alle Scores.

Wenn erkannt:
sofortiger Abbruch der bisherigen Logik
betroffener Layer = Blocker
keine Eskalation
keine Aktivierung
keine Komplexität
Gespräch verlangsamen

7.3 GATE-REGEL
Wenn Gate aktiv:

8. SCHRITT 3: MOVEMENT IMPACT LOGIC (NEU)
8.1 ZIEL
Bestimmt, welcher Layer die Bewegung am stärksten blockiert.

8.2 IMPACT FAKTOREN
Movement Impact ergibt sich aus:
Score-Abweichung
Signalstärke
Layer-Gewichtung
Abhängigkeit zu Handlung

8.3 LAYER GEWICHTUNG
Trust → sehr hoch
Ease → hoch
Momentum → hoch
Clarity → mittel
State → mittel
Authority → situativ
Type → kein Impact
8.4 IMPACT REGEL
Ein Layer wird Blocker, wenn:
→ er die nächste sinnvolle Handlung am stärksten verhindert
 
8.5 EDGE REGEL
Wenn mehrere Layer ähnlich stark sind:
→ wähle den Layer, der näher an Handlung blockiert

Priorität:
Handlung > Verständnis > Wahrnehmung
 
8.6 CONFIDENCE EINFLUSS
Wenn Impact hoch, aber Confidence niedrig:

→ kein harter Blocker
→ Modus = UNCERTAIN
 
9. SCHRITT 4: FINALER BLOCKER
 
Regeln:
genau EIN Blocker
bestimmt Intervention vollständig
andere Layer beeinflussen nur Ausführung

10. SCHRITT 5: CONFIDENCE CHECK
 
Wenn Confidence zu niedrig:
→ keine harte Aussage
→ keine Eskalation
→ nur Klärung / Spiegelung
 
 
11. SCHRITT 6: INTERVENTION
 
Regel:
→ genau eine Intervention
→ keine Kombination
 
Beispiele:
State niedrig → Mirror
State mittel → Reframe
Trust niedrig → Safety / Trust Build
Ease niedrig → Reduce
Clarity niedrig → Structure
Momentum niedrig → Activate
Authority niedrig → Route
